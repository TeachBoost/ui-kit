Tables

The table component is used to display large amounts of data or statistics, lists of users and forms, and anything else that's best displayed by columns and rows. Despite what your 1998 gut is telling you, do not use tables to frame out a page or to position elements—although much easier to position things this way it's not semantically correct and we hold ourselves to the highest of semantic standards. We're not animals.

Basic Tables

To create the basic TeachBoost table simply add the class .table to a <table> element—so simple! Always include <thead> around table headers and <tbody> around the table's data cells. Here's a basic table with it's markup, note that rows have a 1px lightgray border between them:

# Name Email School Role
1 Jerry Campbell jcampbell@schools.tb.com Starfleet Academy Admin
2 Reese Black jrblack@schools.tb.com Rydell High School Teacher 
<!-- Basic table with only one row -->
<table class="table">
    <thead>
        <tr>
            <th>#</th>
            <th>Name</th>
            <th>Email</th>
            <th>School</th>
            <th>Role</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1</td>
            <td>Jerry Campbell</td>
            <td>jcampbell@schools.tb.com</td>
            <td>Starfleet Academy</td>
            <td>Admin</td>
        </tr>
        ...
    </tbody>
</table>

Column Filters and Sorting

Often in complex or long forms it makes sense to provide column filters and sorts. Filters sit below the column header and can be either a text input or dropdown menu. When a column is sortable there will be a clickable caret icon to the right of the header indicating which direction that column is sorted.

Teacher Observation Type Date Status
Reese Black Classroom Walkthrough March 12 In-Progress
Jackie Pillow Informal Observation March 10 Complete

Column sorts can be added simply by dropping in a <span> with a class .column-sort. This will automatically wire up the javascript sorting on that column.

Column filters are added as a second optional <tr> within the <thead> and require the class .filters. You can add either text inputs or select menus in the cells of this row. For inputs use the class .column-filter and a secondary contextual class depending on the type of data it filters: .date, .status, or .name.

For select menus you also must apply the class .column-filter. There are no secondary classes for these menus. The first <option> in the menu must have a value of "-1" and should have no text.

Note: placeholders should be used for all column filter inputs and should generally be the same as the column's header. For instance, for a "Name" column use placeholder="Name" in the <input> and for a form's status use placeholder="Status". Use these for both text inputs and select menus. 
<!-- Table with a filter row and column sorts -->
<table class="table filterable sortable">
    <thead>
        <tr>
            <th>Teacher <span class="caret column-sort"></span></th>
            <th>Observation</th>
            <th>Date</th>
            <th>Status</th>
        </tr>
        <tr class="filters">
            <td><!-- no filter --></td>
            <td>
                <select class="column-filter">
                    <option value="-1"></option>
                    <option value="1">Classroom Walkthrough</option>
                    <option value="2">Informal Observation</option>
                </select>
            </td>
            <td>
                <input type="text" class="column-filter date" />
            </td>
            <td>
                <input type="text" class="column-filter status" />
            </td>
        </tr>
    </thead>
    <tbody>
        ...
    </tbody>
</table>

Zebra Tables

Our true default tables should have zebra-striped rows (alternating white and gray table rows). To create this append the class .table-striped in the <table> element. Rarely should you ever exclude this class as the stripes make a much more readable table—when in doubt stripe the table.

# Name Email School Role
1 Jerry Campbell jcampbell@schools.tb.com Starfleet Academy Admin
2 Reese Black jrblack@schools.tb.com Rydell High School Teacher
3 Fuller Bean fbeank@schools.tb.com Bayside High School Teacher 
<!-- Stripe this table -->
<table class="table table-striped">
    ...
</table>

Condensed Rows

When you need to tighten the table up a bit you can condense the row padding by appending the class .table-condensed to the <table> element. This cuts the table cell padding in half to 4px.

Hover Rows

For a more visual way to indicate at which row the user is looking you can highlight the active row by appending the class .table-hover to the <table> element. This turns on a hover state to every row inside of a <tbody> and is great when you are presenting a lot of data, complex data, long rows, or many (condensed) rows that might be hard to read otherwise.

# Name Email School Role
1 Jerry Campbell jcampbell@schools.tb.com Starfleet Academy Admin
2 Reese Black jrblack@schools.tb.com Rydell High School Teacher
3 Fuller Bean fbean@schools.tb.com Bayside High School Teacher

This works on regular tables and striped tables and will always apply a lightblue hover color. Because of this, beware of the effect if you apply an indicator color to a table row (displayed below). Blue rows will not highlight, for instance.

Contextual Indicators

When building a table you can apply contextual classes to either the entire row in the <tr> or to specific cells in their <td> tag. These classes can be used to indicate a successful or positive action (.success), neutral info (.info), a mild warning (.warning), or an alert (.danger). These work in a striped table and with hover rows.

# Name Email School Role
1 Jerry Campbell jcampbell@schools.tb.com Starfleet Academy Admin
2 Reese Black jrblack@schools.tb.com Rydell High School Teacher
3 Fuller Bean fbean@schools.tb.com Bayside High School Teacher

Add the class .active to a row or to a specific cell to apply a light gray background color. Use this to create the most neutral indicator or to merely call a row or cell out without applying any other context or meaning.
