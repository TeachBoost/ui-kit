Buttons

We use buttons for all application functions that do more than merely open a new page or window (for those we use regular links). Buttons shall be used sparingly and shall always maintain their color:function identity.

Default Primary Success Information Print Warning Danger! Regular link 

<!-- Standard button, used for background or minor actions -->
<a class="gray btn">Default</a>

<!-- Default for submitting forms, initiating actions, and opening modals -->
<a class="blue btn">Primary</a>

<!-- Adding or editing information via a form -->
<a class="green btn">Success</a>

<!-- Contextual button for informational alert messages -->
<a class="teal btn">Information</a>

<!-- Used only for printing -->
<a class="yellow btn">Print</a>

<!-- Indicates caution should be taken with this action -->
<a class="orange btn">Warning</a>

<!-- Indicates a dangerous action (e.g., deleting information) -->
<a class="red btn">Danger!</a>

Using Icons with Buttons

When placing an icon inside of a button there are no additional classes or changes to make other than adding the icon itself. They always appear to the left of the button text and are always the same color as the text. By default there is a 10px right pad on the icon's <i> element; because of this there should be no space between the icon and the text.

Glowing Buttons

The "glowing" button state adds a semi-translucent border to the button and is used primarily in the marketing site and to log in. They are created by adding the .glowing class to the button class stack and work for all colors. These look better on colored backgrounds.

Segment Buttons

Use segmented buttons to toggle between various states or to contextually attach related functions together. It is preferable that one button in a segment always have the "on" state.

First Second Last 

<!-- Blue segmented buttons with text only -->
<menu class="btn-group">
    <a class="first blue btn active" href="javascript:;">First</a>
    <a class="blue btn" href="javascript:;">Second</a>
    <a class="last blue btn" href="javascript:;">Last</a>
</menu>

<!-- Green segmented buttons with icons-->
<menu class="btn-group">
    <a class="first green icon btn" href="javascript:;">
        <i class="fa fa-align-left"></i>
    </a>
    <a class="green icon btn active" href="javascript:;">
        <i class="fa fa-align-center"></i>
    </a>
    <a class="green icon btn" href="javascript:;">
        <i class="fa fa-align-right"></i>
    </a>
    <a class="last green icon btn" href="javascript:;">
        <i class="fa fa-align-justify"></i>
    </a>
</menu>

As you can see segment buttons are created with the <menu> element and class .btn-group. Each button in the group is created like a regular button. When only an icon is present you must add the class .icon. Use the class .active when manually setting an active button.

Note: the .first and .last classes are fallbacks for the preferred :first-child and :last-child pseudo-selectors. They are only required if IE8 support is mandatory and these pseudo-selectors are supported by every modern browser and IE9+.

Button Dropdowns

A dropdown menu is a toggleable context menu to display a vertical list of links. These will typically pop-up from a segmented button with a dropdown caret and provide a great way to reduce a menu system's size and clutter. If you have a series of related items or links that fall under one category or heading these are a good option.

  • Classroom Walkthrough
  • Informal Observation
  • Formal Observation

You can attach a dropdown menu and caret to a button with the segment button classes above. The button that opens the dropdown menu needs class .dropdown-toggle, a data-toggle attribute with value "dropdown", and a <span> inside with class .caret.

The menus are hidden by default and are built from a <ul> with class .dropdown-menu. The menu must be nested inside of a block-level element with position: relative;; this can be done by wrapping the trigger and menu in <div class="dropdown"> but the menu.button-group also works by default if you're using that to open it.

  • Checklist

<!-- Create a segmented button with a dropdown caret -->
<menu class="btn-group">
    <a class="first teal btn" href="javascript:;">Start a New Observation</a>
    <a class="last teal btn dropdown-toggle" href="javascript:;" data-toggle="dropdown">
        <span class="caret"></span>
    </a>
    <ul class="dropdown-menu" role="menu">
        <li><a href="javascript:;">Classroom Walkthrough</a></li>
        <li><a href="javascript:;">Informal Observation</a></li>
        <li><a href="javascript:;">Checklist</a></li>
        <li class="divider"></li>
        <li><a href="javascript:;">Formal Observation</a></li>
    </ul>
</menu>

You can add headers to sections inside the menu by using li.dropdown-header before the menu items. li.divider provides a 2px white divider line and .disabled on the link will visually signify it "off."

The dropdown menu by default positions itself to align with the left of its parent; add the class .dropdown-menu-right to the <ul> to align the dropdown to the right side of its parent. The following shows every option with code:

  • Classroom Walkthrough
  • Informal Observation
  • Bulletin Board Checklist

<!-- Create a dropdown menu and trigger button -->
<div class="dropdown">
    <menu class="button-group">
        <a class="first teal button" href="javascript:;">Start a New Observation</a>
        <a class="last teal button dropdown-toggle" href="javascript:;" data-toggle="dropdown">
            <span class="caret"></span>
        </a>
        <ul class="dropdown-menu dropdown-menu-left" role="menu">
            <li class="dropdown-header">Informals</li>
            <li><a href="javascript:;">Classroom Walkthrough</a></li>
            <li><a href="javascript:;">Informal Observation</a></li>
            <li class="disabled"><a href="#">Bulletin Board Checklist</a></li>
            <li class="divider"></li>
            <li class="dropdown-header">Formals</li>
            <li><a href="javascript:;">Formal Observation</a></li>
        </ul>
    </menu>
</div>

Bootstrap supports submenus but we will never implement a dual-layer menu system in dropdown form in the app. Keep all dropdown menus to one level!

Switch Toggles

Toggles visually represent an "on" state among two available states. Use switch toggles when you want to allow a user to switch only between two application states, such as different "modes" on a tool or different datasets for a chart. They can accept values of On/Off or icons.

On Off

To create a switch toggle, set it up like a button group but add the class .switch-toggle to the <menu> element. Do not use any color classes on the buttons, just .active for the active state and .off for every inactive button in the switch. 

<!-- Toggle switch with icons -->
<menu class="btn-group switch-toggle">
    <a class="first btn off" href="javascript:;"><i class="fa fa-font"></i></a>
    <a class="last btn active" href="javascript:;"><i class="fa fa-table"></i></a>
</menu>
