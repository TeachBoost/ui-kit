Position

We now have a collection of classes that handle most padding, margin, and position styles for block level elements. The Codex speaks of a bygone era in which every element had an individual, unique class with custom definitions for its attributes. This bore a cruel and unweildy beast that only the bravest—or most foolish—of our forefathers could tame.

Today simply chain our new position classnames together for elements instead of creating a custom component-specific class in our stylesheets. The goal here is reusability and not trivial HTML markup reduction.

Padding and Margin

You can add directional padding to an element in increments of 5 from 5 to 40 and then in increments of 10 from 50 to 100. The classname syntax is .{direction}-padding-{#}. For instance, to add a 10px right pad use the class .right-padding-10 and to add bottom padding of 20px use the class .bottom-padding-20.

This syntax is the same for margins as well: for instance, the add a top margin of 25px use the class .top-margin-25 and to add a left margin of 40px use the class .left-margin-40.

There are a few special "group" classes to add or remove all or one side of an element's padding or margins:

Direction Classname Effect
Top .no-top-padding, .no-top-margin Sets the top padding or margin to 0
Right .no-right-padding, .no-right-margin Sets the right padding or margin to 0
Bottom .no-bottom-padding, .no-bottom-margin Sets the bottom padding or margin to 0
Left .no-left-padding, .no-left-margin Sets the left padding or margin to 0
All .no-padding, .no-margin Sets the element's padding or margin to 0

A Note on Headers: we have default margins and no padding set on our block-level headers (<h1>, etc.) that have been calculated for most display cases. We do not create separate classes or subclasses for these elements for various margins/paddings anymore and instead always use a discretionary position class (e.g., <h2 class="top-margin-10">). This significantly reduces class-bloat and you will often use them to push headers (and other components) up and down depending on the margins and/or padding of any elements before or after them on the page. This is fine and expected.

Floats and Clears

Float elements left or right by adding the class .float-left or .float-right respectively. Remove any inherited float by adding the class .float-none.

Clear floated elements by adding the class .clearfix to the element you are floating; this adds a hidden pseudo-selector with a block-level display to clear it in an elegant manner. If you're creating a custom class you can add the LESS class function .clearfix() to build the clear into your new class: 

// new component that clears itself
.newcomponent {
    ...
    float: left;
    .clearfix();
}
Note: you can also manually clear elements by adding <div class="clear"></div>; this is discouraged as it creates an unnecessary element in the dom (and potentially many).

Positioning

Add position values with these respective classes: .relative-position, .absolute-position, and .fixed-position. The absolute and fixed classes carry a default top and left value of 0px. Unless this is what you want, positioning an element absolute or fixed with these classes is generally discouraged as you will often need to assign top, right, bottom, or left numerical offsets anyway.

To help combat this, we do have specific positional classes for the most common values 1px, 2px, 3px, 4px, and 5px in the format .{direction}-{#}. For instance, to set an element 1px from the top and 5px from the left append .top-1 .left-5.

Z-Index

Bootstrap provides default z-index values for different components, and we have set z-indexes for own components in relation to these. These values have been assigned to work together and should be the reference point for new components:

Components Z-Index Value Affected Classes
Navbars, Dropdowns 1000 .navbar, .dropdown-menu
Header, Popovers 1010 <header>, .popover
Tooltips, Fixed navbar 1030 .tooltip, .left-nav, .navbar-fixed-*
Modal background, Fixed buttons 1040 .modal-backdrop, .help.btn
Modals 1050 .modal
