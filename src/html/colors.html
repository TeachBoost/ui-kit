Typography

We use two font families in our brand and application: Signika is used in our logo and for headings/headers and Lucida Grande is used for all regular copy, body content, and plain text. We never use any other fonts anywhere in the application—you only have access to these two fonts :) 

font-family: "Lucida Grande", "Lucida Sans Unicode", "Lucida Sans", 
    Geneva, "Helvetica Neue", Helvetica, Arial, sans-serif;

font-family: Signika, "Trebuchet MS", Tahoma, Verdana, Arial, 
    Helvetica, sans-serif;

Headings

Our first- and second-level headings (<h1> and <h2>) are always set in Signika. Rarely should you ever need to overload one of these headings to change the font to Lucida. Use <h1> headings only for titles on specific documents and printouts, not for any application screen. Use <h2> headings for titles at the top of an application screen or report.

Other headings may be used as needed. Generally, use <h3> headings for subsections on a screen, fieldsets, top level framework domains, or questions inside of an Eagle form. Use <h4> headings for titles or captions on a smaller target section or component. <h5> and <h6> are free-for-all headings to be used at your whimsy.

Body Text

Normal application text is 15px Lucida in #333 with a 20/14 line height. Generate this by applying the class .body to the <p> elements that contain body text. These paragraphs are examples of this text. You should rarely set base application text to less than 15px; we need to maintain this size for readability with our userbase.

Setting Font Attributes

Often you will need to manually set certain font attributes—face, weight, size, style, case, etc. Use the following targeted classes to set these attributes on the element. We do not create custom classes for these attributes in our LESS/CSS files, we string them inside of an element's class attribute in the markup.

Attribute Class Name Effect
Face .sans-font Sets the element's font-family to our Lucida stack
Face .serif-font Sets the element's font-family to our Signika stack (yes I know it's not serif!)
Line Height .lineheight-1-*  Sets the line-height to 1.*em; use 5, 6, 7, or 8.
Color .color-* Sets the font color; use: blue, red, green, purple, yellow, orange, teal, gray, darkgray, seven, five, and  white 
Weight .bold-weight Make the text bold; consider using <strong> instead
Weight .normal-weight Sets the text in normal weight (non-bold)
Style .italicize Sets the text in italics; consider using <em> instead
Spacing .nowrap Removes line-wrapping (the line will not break)
Size .*-text Sets the font-size to 11px (.xs-text), 13px (.sm-text), 15px (.md-text or .body-text), or 17px (.lg-text)
Text Shadow  .text-shadow() Use this mixin to add a text shadow; arguments are horizontal offset, vertical offset, blur, and color (all must be present)
Case .uppercase Sets the text entirely in uppercase

We don't underline text in the application; if you need to apply an emphasis make it bold and/or increase the font size or color. We also have components for calling text out, such as the .alert boxes.
