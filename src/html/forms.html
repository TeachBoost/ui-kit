Forms & Inputs

At our core we build the very best ways to input and gather educator information. "Forms" are the beginning and it's our diligent attention to all of the various inputs—the way we present them and the tools we build on top—that found everything else. As Jeff Bezos said in a letter to his company:

[W]e believe that focusing our energy on the controllable inputs . . . is the most effective way to maximize . . . outputs over time.

—Jeff Bezos, Amazon.com, April 2010

Hey may have been referring to different inputs but the point still stands! We use a number of different inputs within TeachBoost and it is critical to think about and choose the best method for the job.

For the most part, all pre-styled <input>, <textarea>, and <select> elements will automatically receive some styling and control themselves for various devices and implementations. Use our base input class .form-control on elements and wrap input/label bundles in .form-group for spacing.

Types and Usage Guidelines

Text Inputs

Use basic text <input> elements any time the user needs to enter short amounts of variable text. If there is a bounded limit on the types of text to be entered or if it can be grouped, consider using a select menu or option group instead. Generally text inputs should carry a <label> and both should be wrapped in a .form-group.

Note: input labels should always be in title case and end with a colon. Apply the class .control-label to them when with a form, like here. 
<!-- Basic text input with a label -->
<div class="form-group">
    <label class="control-label" for="inputField">Input Label:</label>
    <input type="text" 
           class="form-control" 
           placeholder="Placeholder" 
           id="inputField" />
</div>

Textareas

Use <textarea> fields to allow users to enter large amounts of text, notes, feedback, and generally anything that might be longer than 100 characters. Use the same .form-group wrapper and element class, .form-control. As a fallback, always include a value for the columns (number of horizontal characters) and rows (vertical lines). Defaults should be 50 and 5. 

<!-- Basic textarea (wrapper HTML removed) -->
<textarea cols="50" 
          rows="5" 
          class="form-control" 
          placeholder="Placeholder" 
          id="textareaField"></textarea>

Note also that the element requires a separate end tag and that the content goes between those tags.

Select Menus

Use select menus when you have a predefined list of options or answer choices that is too long to display on the screen in a checkgroup or radiogroup. Use the same .form-group wrapper and element class, .form-control, and <select> elements should also have a placeholder. 

<!-- Basic select menu (wrapper HTML removed) -->
<select id="selectField"
        class="form-control"
        placeholder="Select options">
    <option value="1">Option Choice 1</option>
    <option value="2">Option Choice 2</option>
    ...
    <option value="10">Option Choice 10</option>
</select>

If you want to pre-select an option in a select menu use the selected attribute at the end of the <option> element. There is no reason to add a value for this attribute.

Radio Buttons and Checkboxes

Use radio buttons when the user must select one choice from a limited set of values. Use checkboxes when the user may select one or more choices from a set of values. The default behavior of both inputs is to stack them vertically: 

<!-- Checkbox group with two selected -->
<div class="checkbox">
    <label>
        <input type="checkbox" value="1" /> Checkbox option one
    </label>
</div>
<div class="checkbox">
    <label>
        <input type="checkbox" value="2" checked /> Checkbox option two
    </label>
</div>
<div class="checkbox">
    <label>
        <input type="checkbox" value="3" checked /> Checkbox option three
    </label>
</div>

<!-- Radio button group with one selected -->
<div class="radio">
    <label>
        <input type="radio" name="radios" id="radio1" value="1" checked />
        Radio button one
    </label>
</div>
<div class="radio">
    <label>
        <input type="radio" name="radios" id="radio2" value="2" />
        Radio button two
    </label>
</div>
<div class="radio">
    <label>
        <input type="radio" name="radios" id="radio3" value="3" />
        Radio button three
    </label>
</div>

Notice that each form input is contained inside of its own contextual <div> (use .checkbox and .radio respectively) rather than the .form-group container class. The <label> element wraps the form input as well making the input text clickable. Options within a checkbox group or radio button group should all have the same name attribute

To pre-select an option within one of these groups simply add the attribute checked to the end of the input. As with the select menu, there is no reason to add a value for this attribute and it will work cross-browser.

File Inputs

Use a file input when the user must upload a local file. This field is highly variable style-wise among browsers and for security reasons it is impossible to style without using a javascript solution. Currently we rely on the browser's native styling. 

<!-- Default file input -->
<div class="form-group">
    <label class="control-label" for="fileInput">File Input:</label>
    <input type="file" id="fileInput" />
</div>

Input Sizing

We have two additional size classes you can apply to reduce the padding, font size, and height of the <input>. Use .input-sm to make it smaller and .input-lg to make it bigger. So easy!

Disabled Inputs

You can disable a form input by adding the disabled attribute to the input element. Currently <input>, <select>, <button>, and <textarea> elements are supported. To globally disable form inputs you must wrap the elements in a <fieldset> with the disabled. attribute. Note this does not work for IE9 and down, so the preferred way is to do both (regrettably).

   
    
<!-- Disabled inputs using the <fieldset> and invidual method-->
<fieldset disabled>
    <div class="form-group">
        <input class="form-control" placeholder="Disabled input" disabled />
    </div>
    <div class="form-group">
        <select class="form-control" placeholder="Disabled select" disabled>
            ...
        </select>
    </div>
    <div class="form-group">
        <textarea class="form-control" cols="25" rows="4" placeholder="Disabled textarea" disabled>
        </textarea>
    </div>
    <div class="checkbox no-top-margin">
        <label><input type="checkbox" value="1" disabled /> Disabled checkbox</label>
    </div>
    <div class="radio">
        <label><input type="radio" value="2" disabled /> Disabled radio</label>
    </div>
    <div class="form-group">
        <button class="blue btn" type="submit" disabled>Disabled Button</button>
    </div>
<fieldset>

Validation States

We offer a suite of "validation states" to add additional visual style and contextual information on our form inputs based on different user interactions: success (green), warning (yellow), and error (red). Apply these to relevant inputs by appending the respective class to the .form-group or other wrapper element: .has-success, .has-warning, .has-error. This will affect not only the input element but also any label or sub-text. 

<!-- Inputs with validation colorz -->
<div class="form-group has-success">
    <label class="control-label" for="inputSuccess">Input with success</label>
    <input type="text" class="form-control" id="inputSuccess">
</div>
<div class="form-group has-warning">
    <label class="control-label" for="inputWarning">Input with warning</label>
    <input type="text" class="form-control" id="inputWarning">
</div>
<div class="form-group has-error">
    <label class="control-label" for="inputError">Input with error</label>
    <input type="text" class="form-control" id="inputError">
</div>

Laying Out Forms

When constructing a form and laying it out on screen you should use the same scaffolding grid system that we use throughout the app. Because <input> elements themselves have a default width of 100% their horizontal size is controlled by their wrapper elements. Use the correct grid classes on a wrapper <div> to control for sizing here; this will most likely just be a .col-xs-* to handle all screen sizes. 

<!-- Laying out a form to span 3 weird columns 100% -->
<form action="self.php" method="post">
    <div class="row">
        <div class="col-xs-2 form-group">
            <input type="text" class="form-control" placeholder=".col-xs-2" />
        </div>
        <div class="col-xs-3 form-group">
            <input type="text" class="form-control" placeholder=".col-xs-3" />
        </div>
        <div class="col-xs-7 form-group">
            <input type="text" class="form-control" placeholder=".col-xs-7" />
        </div>
    </div>
</form>

Help Text

When you need to provide additional information on a specific form element, often after validating user input, you can display a message inside of the <span.help-block> element. This should sit below the form input and inside the same .form-group.

Enter the teacher's name as it appears on their official personnel record at your school. 
<!-- Insert some help text below a text input -->
<div class="col-md-6 form-group">
    <label class="control-label">Teacher Name:</label>
    <input type="text" class="form-control" placeholder="Teacher Name" />
    <span class="help-block">
        Enter the teacher's name as it appears on their official personnel record at your school.
    </span>
</div>
